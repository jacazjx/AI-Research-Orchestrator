## Overview

This document provides detailed scoring rubrics for evaluating hypothesis quality across seven criteria. Each criterion is scored on a 1-5 scale, with clear guidelines for each score level.

**Total Score Range**: 7-35 points

| Score Range | Interpretation |
|-------------|----------------|
| 31-35 | Excellent hypothesis - ready for testing |
| 26-30 | Strong hypothesis - minor refinements possible |
| 21-25 | Adequate hypothesis - significant work needed |
| 16-20 | Weak hypothesis - major revision required |
| 7-15 | Inadequate hypothesis - reconsider approach |

## Testability

**Definition**: The extent to which a hypothesis can be empirically tested through observation or experimentation.

**Key Question**: Is there a feasible experiment that could test this hypothesis?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Clear, feasible experiment with available methods | "Increasing learning rate by 2x will reduce convergence time by 30%" - can be directly tested |
| 4 | Good | Testable with minor methodological development | Testable with slight adaptation of existing protocols |
| 3 | Adequate | Testable but requires significant method development | Requires new measurement techniques but feasible |
| 2 | Weak | Difficult to test with current methods | Requires substantial technological advances |
| 1 | Poor | Not testable with any known methods | Purely theoretical or metaphysical claim |

### Evaluation Checklist

- [ ] Can variables be operationalized?
- [ ] Are measurement methods available?
- [ ] Is sample size achievable?
- [ ] Can the experiment be completed in reasonable time?
- [ ] Are resources available or obtainable?

## Falsifiability

**Definition**: The extent to which a hypothesis can be proven false through empirical evidence.

**Key Question**: What specific evidence would disprove this hypothesis?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Clear falsification criteria with specific outcomes | "If accuracy decreases with larger models, hypothesis is falsified" |
| 4 | Good | Falsifiable with well-defined boundary conditions | Multiple clear disconfirming patterns identified |
| 3 | Adequate | Falsifiable but criteria somewhat vague | Can be falsified but requires interpretation |
| 2 | Weak | Difficult to falsify due to vague predictions | Many ad hoc explanations possible |
| 1 | Poor | Not falsifiable - compatible with any outcome | Tautological or irrefutable |

### Evaluation Checklist

- [ ] Are there specific outcomes that would disprove it?
- [ ] Are falsification criteria stated before testing?
- [ ] Can the hypothesis be cleanly rejected?
- [ ] Are there clear boundary conditions?
- [ ] Is the hypothesis specific enough to be falsified?

## Parsimony

**Definition**: The principle that the simplest explanation that accounts for the data is preferred (Occam's Razor).

**Key Question**: Does the hypothesis avoid unnecessary complexity?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Minimal assumptions, elegant explanation | Single mechanism explains all observations |
| 4 | Good | Few assumptions, some complexity justified | 2-3 well-justified assumptions |
| 3 | Adequate | Moderate complexity, partially justified | Multiple assumptions, some unjustified |
| 2 | Weak | Many assumptions, complexity not justified | Rube Goldberg-like explanation |
| 1 | Poor | Extremely complex, many unfounded assumptions | Requires extensive special cases |

### Evaluation Checklist

- [ ] How many assumptions does it make?
- [ ] Is each assumption necessary?
- [ ] Could a simpler explanation account for the same data?
- [ ] Are there unnecessary entities or mechanisms?
- [ ] Is the core mechanism straightforward?

## Explanatory Power

**Definition**: The extent to which a hypothesis explains the observed phenomena and related findings.

**Key Question**: How much does this hypothesis explain?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Explains all key observations and predicts new phenomena | Unifies multiple disparate findings |
| 4 | Good | Explains most observations, some predictive power | Accounts for primary findings well |
| 3 | Adequate | Explains some observations, limited prediction | Partial explanation of phenomenon |
| 2 | Weak | Explains few observations, no prediction | Very narrow explanatory scope |
| 1 | Poor | Does not explain key observations | Fails to account for primary data |

### Evaluation Checklist

- [ ] Does it explain the original observation?
- [ ] Does it account for related findings?
- [ ] Does it explain contradictory evidence?
- [ ] Does it generate novel predictions?
- [ ] Does it unify previously separate phenomena?

## Scope

**Definition**: The range of situations, domains, or contexts to which the hypothesis applies.

**Key Question**: How widely does this hypothesis apply?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Broad applicability across domains/contexts | Applies to all machine learning architectures |
| 4 | Good | Clear applicability within a domain | Applies to transformer-based models generally |
| 3 | Adequate | Applicable to specific subset | Applies to NLP transformers under specific conditions |
| 2 | Weak | Very narrow applicability | Applies only to one specific model configuration |
| 1 | Poor | Ad hoc explanation, no generalizability | Applies only to the specific instance observed |

### Evaluation Checklist

- [ ] To what domains does it apply?
- [ ] What are the boundary conditions?
- [ ] Can it generalize to related phenomena?
- [ ] Is it too narrow or too broad?
- [ ] Are scope limitations clearly stated?

## Consistency

**Definition**: The extent to which a hypothesis aligns with established knowledge and theories.

**Key Question**: Does the hypothesis contradict established findings?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Fully consistent with established knowledge | Extends current theory naturally |
| 4 | Good | Consistent with minor tensions that can be resolved | Minor contradictions that are explainable |
| 3 | Adequate | Some contradictions that require resolution | Requires modification of existing views |
| 2 | Weak | Significant contradictions with established findings | Challenges core assumptions without evidence |
| 1 | Poor | Contradicts well-established facts | Incompatible with proven results |

### Evaluation Checklist

- [ ] Does it contradict established theories?
- [ ] Does it align with empirical evidence?
- [ ] Are contradictions justified?
- [ ] Does it require revising accepted views?
- [ ] Are there internal logical contradictions?

## Novelty

**Definition**: The extent to which a hypothesis provides new insights or perspectives beyond existing explanations.

**Key Question**: Does this hypothesis advance our understanding?

### Scoring Guide

| Score | Level | Description | Example |
|-------|-------|-------------|---------|
| 5 | Excellent | Paradigm-shifting insight, opens new research direction | Novel mechanism never proposed before |
| 4 | Good | Significant new perspective on existing problem | New combination of known mechanisms |
| 3 | Adequate | Incremental advance over existing explanations | Minor refinement of existing theory |
| 2 | Weak | Largely restates existing explanations | Marginal contribution |
| 1 | Poor | No new insights, already well-established | Already widely accepted view |

### Evaluation Checklist

- [ ] Is this hypothesis new?
- [ ] Does it offer a different perspective?
- [ ] What does it add to existing knowledge?
- [ ] Is it a significant departure from current views?
- [ ] Would accepting it change the field?

## Scoring Guide

### Score Assignment Process

1. **Read the hypothesis carefully** - Ensure full understanding
2. **Apply each criterion independently** - Avoid halo effects
3. **Consult the rubric** - Match to score descriptions
4. **Document justification** - Explain why the score was assigned
5. **Check for bias** - Ensure consistent standards across hypotheses

### Common Scoring Errors

| Error | Description | Prevention |
|-------|-------------|------------|
| Halo Effect | High scores in one area bias others | Score each criterion independently |
| Central Tendency | Avoiding extreme scores | Use full range when justified |
| Leniency/Severity | Overall too harsh or easy | Compare to rubric, not other hypotheses |
| Recency Bias | Over-weight recent information | Consider all evidence equally |

### Inter-Rater Reliability

When multiple raters evaluate hypotheses:

1. **Calculate agreement**: Use Cohen's kappa or ICC
2. **Discuss discrepancies**: Identify source of disagreement
3. **Calibrate**: Align on scoring standards
4. **Document**: Record consensus scores and reasoning

Target: ICC > 0.75 for adequate reliability
